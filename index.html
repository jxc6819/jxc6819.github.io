This project is a fully custom VR level mod for I Expect You To Die 2, built from scratch and integrated directly into the live game. The level and puzzles were carefully designed and iterated on to guide the player naturally through the space while encouraging experimentation and problem solving. Much of the work focused on making multiple systems interact reliably, handling edge cases and failure states, and refining level flow and feedback so the mechanics felt intuitive in actual play. Overall, this project highlights how I apply technical gameplay systems to level design in a way that supports clarity, pacing, and player understanding.

Baseball VR Game

This project is an unfinished baseball VR game developed as my high school senior capstone project. Over the course of about two months, I worked on building a playable, physics-driven baseball experience that allowed the player to alternate between batting and fielding. From the batting side, the game features realistic ball physics, a functional interactive baserunning system, and AI-controlled fielders. From the fielding side, the player can field ground balls, fly balls, pop-ups, and one-hoppers, with basic aim assist implemented for throwing. From a technical standpoint, this project focused heavily on physics-based gameplay and system coordination. I implemented ball physics, batting logic, baserunner behavior, basic AI decision-making, and simple inning simulations to handle play between user-controlled moments. Many systems were built quickly and iterated on under tight time constraints, which made the project a valuable learning experience in balancing scope, functionality, and stability. I worked on this project with a partner, where I handled all programming and gameplay logic while they focused on modeling and assets. Automated Hospital Survey Data Processor

This project is a Python automation I built for a hospital-run Narrative Medicine program to handle participant survey data. Previously, staff had to manually open pre-surveys, post-surveys, and attendance logs and then copy values into a master spreadsheet by hand, one person at a time. My script pulls data from all of those sources automatically, lines up participants across spreadsheets, converts written survey responses into consistent numeric values, and fills out the master sheet in a single run. It handles real-world messiness like participants completing only a pre-survey or post-survey, surveys with columns in different orders, incomplete responses, and attendance tracked as repeated entries instead of totals.

From a development standpoint, this project shows how I approach building systems that actually get used. I designed it to be flexible rather than brittle, using fuzzy matching and defensive checks so it could adapt as forms changed or data came in imperfectly. The end result turned what used to be 30–40 hours of repetitive manual work into a process that runs in about two minutes, while also reducing errors. It reflects my focus on understanding the full workflow, anticipating edge cases early, and building tools that are reliable, practical, and easy for non-technical users to rely on.

Mycord TCP Chat Client

This project is a custom client for the mycord chat service written in C for Linux and Unix systems. The client establishes a TCP connection to a remote server and implements a fixed-size binary message protocol to handle user login, message sending, message receiving, system notifications, and server disconnects. It supports command line configuration for IP, port, and domain name resolution, performs DNS lookup when connecting by domain, and formats incoming messages with timestamps, ANSI color highlighting, and audible alerts for user mentions. The application uses pthreads to concurrently receive network messages while reading user input from STDIN, allowing for real-time interaction without blocking.

From a technical perspective, this project demonstrates my ability to work close to the system and reason about networking, concurrency, and robustness under real constraints. I implemented socket communication, protocol parsing, input validation, and multithreaded coordination without relying on high-level abstractions, while carefully handling error cases to prevent invalid states or server disconnects. The project required designing around blocking I/O, coordinating clean shutdown across threads, and handling signals and EOF gracefully. Overall, this work reflects my comfort with low-level systems programming, defensive coding practices, and building reliable software that interacts directly with the operating system and the network.
