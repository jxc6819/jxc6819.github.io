Mod for 'I Expect You To Die' VR Game

This project is a fully custom VR level mod for I Expect You To Die 2, built from scratch and integrated directly into the live game. The player infiltrates an enemy submarine and must sabotage it from the inside by solving interconnected environmental puzzles, operating physical controls, and reacting as systems begin to fail. The level and puzzles were carefully designed and iterated on to guide the player naturally through the space while encouraging experimentation and problem solving. Player actions directly affect what happens next, creating a tense, hands-on experience driven by clear cause and effect.

From a development perspective, this project demonstrates my ability to build complex gameplay systems under real constraints. I worked in Unity and C# within an IL2CPP environment, creating custom tools and runtime logic without editor access or source code. Much of the work focused on making multiple systems interact reliably, handling edge cases and failure states, and refining level flow and feedback so the mechanics felt intuitive in actual play. Overall, this project highlights how I apply technical gameplay systems to level design in a way that supports clarity, pacing, and player understanding.

Baseball VR Game

This project is an unfinished baseball VR game developed as my high school senior capstone project. Over the course of about two months, I worked on building a playable, physics-driven baseball experience that allowed the player to alternate between batting and fielding. From the batting side, the game features realistic ball physics, a functional interactive baserunning system, and AI-controlled fielders. From the fielding side, the player can field ground balls, fly balls, pop-ups, and one-hoppers, with basic aim assist implemented for throwing. While unpolished and occasionally janky, the project reached a point where a full playable exhibition-style game was possible.

From a technical standpoint, this project focused heavily on physics-based gameplay and system coordination. I implemented ball physics, batting logic, baserunner behavior, basic AI decision-making, and simple inning simulations to handle play between user-controlled moments. Many systems were built quickly and iterated on under tight time constraints, which made the project a valuable learning experience in balancing scope, functionality, and stability. I worked on this project with a partner, where I handled all programming and gameplay logic while they focused on modeling and assets. Overall, this project reflects an important step in my growth as a developer and shows how I approached complex, system-heavy gameplay problems earlier in my development.
